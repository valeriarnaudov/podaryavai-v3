# 🛠️ Локално Стартиране и Тестване (Local Setup Guide)

Този файл описва стъпките, необходими за стартиране на проекта на вашия компютър (локално) с цел тестване и разработка, преди да го качите в продукция.

---

## 1. Изисквания (Prerequisites)

Уверете се, че имате инсталирани следните програми на вашия компютър:
- [Node.js](https://nodejs.org/) (Препоръчително версия 18 или по-нова)
- [Git](https://git-scm.com/)
- [Docker](https://www.docker.com/) (Необходимо **само** ако искате да стартирате цялото Supabase ядро локално на вашата машина. Ако ползвате облачния Supabase проект за база данни, Docker не ви трябва).

## 2. Инсталиране на Зависимостите

Отворете терминала, навигирайте до папката на проекта (`/Users/valto/Dev/podaryavai v3`) и изпълнете:
```bash
npm install
```
*Това ще свали всички нужни пакети като React, Tailwind CSS, Framer Motion, i18next, Stripe, Supabase JS клиент и др.*

## 3. Настройка на Environment Variables (.env)

За да може локалното приложение да се свърже с вашата база данни, ви трябва `.env` файл. Създайте файл с име `.env` в основната папка (до `package.json`) и добавете следните редове:
```env
VITE_SUPABASE_URL="https://<вашият-проект>.supabase.co"
VITE_SUPABASE_ANON_KEY="<вашият-anon-key>"
```
*(Тези ключове можете да намерите във вашия Supabase Dashboard -> Project Settings -> API).*

## 4. Стартиране на Локалния Сървър (Frontend)

За да стартирате React приложението локално, използвайте командата:
```bash
npm run dev
```
Обикновено сървърът ще се стартира на `http://localhost:5173`. Ако искате да го тествате **през телефона си** (докато сте на същия Wi-Fi), използвайте:
```bash
npm run dev -- --host
```
*Ще ви се покаже IP адрес (напр. `http://192.168.1.5:5173`), който можете да отворите в браузъра на телефона си, за да тествате PWA усещането, Landing страницата, тъмната тема и анимациите на Swipe.*

---

## 5. Тестване на Supabase Edge Функциите локално (AI, Stripe, Notifications)

Edge функциите (Gemini, Stripe, Daily Notifications) се изпълняват на сървър/Edge node. За да ги тествате локално без да ги качвате веднага в облака, използвайте Supabase CLI:

**Стъпка 5.1:** Добавете вашите тайни (APIs) в локален `.env` файл вътре в папка `supabase/` (не главната папка, а вътре в `supabase` папката, създайте копие на `.env`). Пример за `supabase/.env`:
```env
GEMINI_API_KEY="AIzaSy..."
STRIPE_SECRET_KEY="sk_test_..."
STRIPE_WEBHOOK_SECRET="whsec_..."
SUPABASE_URL="https://<вашият-проект>.supabase.co"
SUPABASE_SERVICE_ROLE_KEY="<вашият-service-role-key>"
```

**Стъпка 5.2:** Стартирайте локалния сървър за функции (изисква инсталиран `npx` и Docker):
```bash
npx supabase functions serve --env-file ./supabase/.env
```

**Важно за фронтенда при локално тестване на функциите:** По подразбиране фронтендът (`supabase.functions.invoke()`) ще търси функциите в облака. За да го накарате да сочи към локалния ви компютър по време на разработка, временно разкоментирайте реда в `src/lib/supabase.ts` (или където ги викате), за да добавите локалния url, но обикновено локалният Vite сървър с облачната база е напълно достатъчен за UI/UX тестове.

## 6. Какво да тествате локално?
- **Landing Page, Theme & i18n:** Проверете дали превключвателят за Dark/Light Mode работи, превключете езика (BG/EN) и вижте дали всички текстове по екрана се превеждат в реално време (чрез `useTranslation()`).
- **PWA Инсталация:** Отворете локалния линк през Chrome на телефона си и натиснете "Add to Home Screen".
- **Swipe Animations:** Влезте в екрана "Giftinder" и тествайте плъзгането на картите.
- **Calendar Sync (.ics):** Отидете на Home екрана и натиснете "Sync to Calendar", за да проверите дали файлът с .ics се генерира правилно.
- **UI Admin Logs:** Тествайте таб `Karma Purchases` в Admin панела с реална дата.
